pfctl 常用命令參考

本文整理了在 macOS 環境下使用 pfctl（Packet Filter）的常見命令、參數說明及典型使用場景。

1. 基礎概念

  • Anchor（錨點）：pf 規則的邏輯分組概念，類似 iptables 的表+鏈組合。
  • Table（表格）：pf 中的高效地址集合，常用於白名單/黑名單。
  • Rule（規則）：由匹配條件與動作組成，按 /etc/pf.conf 順序執行。
  • 動作：pass（放行）、block（阻斷）、match、scrub。
  • 方向：in（入站）、out（出站）。

macOS pf 不支援按行號插入/刪除規則，須透過編輯設定檔後重新載入。

2. 命令結構

pfctl [參數] [選項]
參數說明
-s顯示資訊，搭配 rules、nat、all 等。
-f載入指定設定檔（覆蓋所有現有規則）。
-e啟用封包過濾。
-d停用封包過濾。
-N不載入 NAT 規則（與 -f 搭配）。
-a指定 anchor。
-t操作 table，搭配 -T 子命令。
-v詳細輸出。

3. 常用命令速查

3.1 查看狀態與規則

pfctl -s rules -v
pfctl -s nat
pfctl -s all
pfctl -s rules -a myanchor
pfctl -s queue

3.2 啟用 / 停用

sudo pfctl -e
sudo pfctl -d
sudo pfctl -f /etc/pf.conf
sudo pfctl -nf /etc/pf.conf

3.3 管理 Tables

pfctl -s tables
pfctl -t blacklist -T show
pfctl -t blacklist -T add 10.0.0.1
pfctl -t blacklist -T delete 10.0.0.1
pfctl -t whitelist -T load -f /etc/whitelist.txt
pfctl -t blacklist -T test 10.0.0.1

3.4 查看連接狀態

pfctl -s state
pfctl -s state -v
pfctl -s info

4. pf.conf 語法速查

action [direction] [log] [quick] [on interface] [proto protocol] [from src] [to dst] [port ...]

4.1 基本放行與阻斷

pass in quick on lo0 all
block in quick from 10.0.0.0/8 to any
pass in proto tcp from 192.168.1.0/24 to any port 22
block in all

4.2 狀態追蹤

pass out proto tcp all keep state
pass in proto tcp all modulate state
pass in proto tcp to any port { 80, 443 } keep state

4.3 NAT / 端口轉發

nat on en0 from 192.168.0.0/24 to any -> (en0)
rdr pass on en0 proto tcp to any port 2222 -> 10.0.0.10 port 22
rdr pass on lo0 proto tcp to any port 80 -> 127.0.0.1 port 8080

4.4 Table 整合

table <blacklist> persist
block drop in quick from <blacklist> to any

4.5 LOG 記錄

block in log all
pass in log proto tcp to any port 22

5. 場景示例

場景 A：僅允許特定網段 SSH

# /etc/pf.conf
block in all
pass out all keep state
pass in quick on lo0 all
pass in proto tcp from 192.168.1.0/24 to any port 22 keep state

場景 B：macOS 共享 Internet（NAT）

# /etc/pf.conf
nat on en0 from 192.168.2.0/24 to any -> (en0)
pass in on en1 from 192.168.2.0/24 to any keep state
pass out on en0 from 192.168.2.0/24 to any keep state

6. 故障排查建議

  1. 所有 pfctl 操作需 sudo，確認使用者有 sudo 權限。
  2. 使用 pfctl -nf /etc/pf.conf 預檢語法。
  3. 規則按順序匹配，quick 關鍵字可中斷後續規則評估。
  4. macOS 升級後 /etc/pf.conf 可能被覆寫，請備份。
  5. tcpdump -r /var/log/pflog 查看 pf 日誌。
